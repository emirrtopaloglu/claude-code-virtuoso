# User Journeys Directory

This directory stores user journey documentation created by the `/vision` skill.

## Purpose

User journeys map out the step-by-step experience of users interacting with the product, ensuring smooth and intuitive workflows.

## Template

When documenting a user journey, include:

```markdown
# [Journey Name]

## Persona
[Which persona this journey applies to]

## Scenario
[Context: When does this journey occur?]

## Steps
1. [Step 1] - [User action] → [System response]
2. [Step 2] - [User action] → [System response]
3. [Step 3] - [User action] → [System response]
...

## Happy Path
[The ideal user flow]

## Edge Cases
- [What could go wrong?]
- [How do we handle errors?]
- [Alternative paths]

## Success Criteria
- [How do we know the journey was successful?]
- [What metrics matter?]
```

## Usage

Journeys are created during `/vision` product discovery and used to:
- Design intuitive user flows
- Identify potential friction points
- Ensure comprehensive feature coverage
- Guide testing scenarios

## Example Journey Types

- **Onboarding Journey** - First-time user experience
- **Core Feature Journey** - Primary use case
- **Recovery Journey** - Error handling and resolution
- **Advanced Feature Journey** - Power user workflows
